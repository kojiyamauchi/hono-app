# Review Response Skill

## 概要

GitHub Pull Requestに入ったレビュー指摘へ対応し、修正・確認・PR本文更新まで行うスキルです。

レビューを行う側の手順は `.claude/skills/pr-review/SKILL.md` を使用し、このSkillではレビューに対応する側の作業に集中します。

## 使用タイミング

- ユーザーが「レビュー対応して」と依頼した時
- ユーザーがPR番号やURLを指定して、レビュー指摘への対応を求めた時
- レビューコメント対応後に、PR本文・差分・コミット履歴の整合性確認を求められた時

## 参照する正本

- レビュー対応時のPR本文更新ルールは [CLAUDE.md](../../../CLAUDE.md) の `#### レビュー対応時のPR本文更新` を正本とする
- レビュー対応コミット形式は [CLAUDE.md](../../../CLAUDE.md) の `#### レビュー指摘への対応コミット` に従う
- コミット粒度、PRタイトル、ブランチ命名、push方法も [CLAUDE.md](../../../CLAUDE.md) に従う

## 実行手順

### 1. PRとレビューコメントの確認

```bash
gh pr view <PR番号またはURL>
gh pr checks <PR番号>
gh pr diff <PR番号>
```

- 対象PRの番号、base/head branch、最新HEADを確認する
- CI状態を確認する
- 未解決のレビューコメント、レビュー本文、通常コメントを確認する
- 指摘がインラインコメントの場合は、対象ファイルと行を確認する

### 2. 対応方針の整理

- 指摘ごと、または関心事ごとに対応単位を分ける
- 修正が必要な指摘と、説明・コメント返信で足りる指摘を分ける
- 指摘同士が衝突する場合は、作業前にユーザーへ確認する

### 3. 修正の実装

- 既存のアーキテクチャ、features設計、命名規則に合わせて修正する
- 不要なリファクタリングを混ぜない
- レビュー指摘に対応する変更だけを行う

### 4. 確認コマンドの実行

変更内容に応じて必要な確認を実行する。

```bash
bun run spellcheck
bun run lint
bun run typecheck
bun test
bun run build
```

ドキュメントのみの変更でも、少なくとも `bun run spellcheck` を確認する。

### 5. レビュー対応コミットの作成

レビュー指摘へ対応する追加コミットは、[CLAUDE.md](../../../CLAUDE.md) の `#### レビュー指摘への対応コミット` の形式に従う。

- 指摘が複数ある場合は、指摘ごと、または関心事ごとにコミットを分ける
- 各コミット直前に `.claude/skills/commit-granularity/SKILL.md` をSkillとしてinvokeし、「コミット直前チェック」でステージ済み差分の粒度を確認する

### 6. PR本文の更新

push前またはpush直後に、PR本文を最新状態へ更新する。

PR本文更新の詳細項目は [CLAUDE.md](../../../CLAUDE.md) の `#### レビュー対応時のPR本文更新` に従う。

確認には必要に応じて以下を使う。

```bash
gh pr diff <PR番号>
gh pr view <PR番号> --json commits
```

### 7. pushと最終確認

```bash
git push <push先> <branch>
gh pr checks <PR番号>
```

- push後にCI状態を確認する
- PR本文、差分、コミット履歴が現在のPR状態を正しく説明しているか確認する
- 必要に応じてレビュー対応コメントを投稿する

## 注意事項

- PR本文更新は、レビュー対応フローの一部として必ず扱う
- PR本文の詳細ルールをSkill内に重複させず、CLAUDE.mdを参照する
- `pr-review` Skillと混同せず、レビューする側ではなく対応する側の作業に使う
- コメント、ドキュメント、コミットメッセージは日本語ルールに従う
